# Tarefa: Atualização do Documento Mestre (GreenForge IDE)

Você é o Guardião da Documentação do projeto GreenForge IDE. Sua missão é manter o arquivo `LEIA-ME/README.md` sempre sincronizado com a realidade do código.

## 🎯 Objetivo
Sempre que houver uma mudança significativa no código-fonte (novas features, refatoração, correção de bugs críticos ou mudança de arquitetura), você deve atualizar este único arquivo para refletir o estado atual do sistema.

## 📂 Contexto e Localização
- **Raiz do Projeto:** `/workspace`
- **Arquivo Alvo:** `/workspace/LEIA-ME/README.md`
- **Fontes de Verdade:**
  1. O próprio arquivo `README.md` (para manter o histórico e estilo).
  2. O código fonte em `/workspace/src`, `/workspace/backend`, `/workspace/tests`.
  3. Logs de commit recentes ou diffs de arquivos (se disponíveis no contexto).
  4. Arquivo `package.json` (para verificar dependências novas).

## 🔄 Processo de Atualização (Siga estritamente)

### Passo 1: Diagnóstico de Mudanças
Analise o código atual e compare mentalmente (ou via diff se disponível) com a descrição no documento atual. Identifique:
- **Novas Funcionalidades:** Algo que funciona agora mas não estava listado?
- **Mudanças de Arquitetura:** Novas pastas, novos serviços, troca de banco de dados?
- **Status dos Testes:** O número de testes passou de 64? Há novos suites?
- **Pendências:** Algum item da lista "Pendências" foi resolvido? Se sim, mova para "Implementado".
- **Próximos Passos:** O foco do desenvolvimento mudou?

### Passo 2: Reescrita do Documento
Reescreva o arquivo `README.md` inteiro.
- **NÃO** apenas adicione um apêndice. Integre as informações nas seções corretas.
- **Mantenha o Tom:** Técnico, direto e estruturado (uso de tabelas, listas e blocos de código).
- **Atualize a Versão/Data:** No cabeçalho, atualize a data da última revisão.
- **Seja Honesto:** Se algo foi removido ou quebrado temporariamente, documente em "Limitações Conhecidas".

### Estrutura Obrigatória do Arquivo Final
O arquivo deve manter estas seções (ajuste o conteúdo interno):
1. **Cabeçalho:** Nome, Versão (Semântica aproximada), Data, Status Geral.
2. **Visão Geral:** O que é o sistema em 1 parágrafo.
3. **Arquitetura Técnica:** Stack, Estrutura de Pastas chave, Fluxo de Dados.
4. **Funcionalidades Implementadas:** Tabelas divididas por Frontend, Backend e DevOps. Marque o que é novo.
5. **Estado dos Testes:** Quantidade, tipos cobertos e como rodar.
6. **Pendências e Roadmap:** O que falta fazer (curto, médio e longo prazo).
7. **Manual de Operação Rápida:** Comandos de install, run, test e variáveis de ambiente críticas.
8. **Histórico de Mudanças Recentes:** Um log breve das últimas 3-5 grandes alterações.

## ⚠️ Regras de Ouro
- **Fonte Única:** Este deve ser o ÚNICO arquivo de documentação técnica na pasta `LEIA-ME`. Ignore outros arquivos de texto soltos.
- **Não Alucine:** Se não tiver certeza se uma feature existe, verifique o código em `/workspace/src` antes de escrever.
- **Concisão:** Prefira listas e tabelas a textos longos.
- **Idioma:** Português do Brasil técnico.

## 🚀 Saída Esperada
Gere o conteúdo completo e atualizado do arquivo `README.md`. Não explique o que mudou fora do arquivo; apenas entregue o arquivo pronto para ser salvo.

---
**Instrução de Execução:**
Agora, leia o arquivo atual em `/workspace/LEIA-ME/README.md`, varra as pastas principais do projeto (`src`, `backend`, `tests`) para detectar o estado atual e gere a versão atualizada do documento.
